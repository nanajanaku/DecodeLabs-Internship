# 📈 Week 4: Data Visualization
### Decode Lab Data Analytics Internship

---

## 📌 Project Overview
The final week focused on bringing the data to life 
through visual representations. I created an interactive 
dashboard in Power BI that communicates key insights 
clearly and professionally using the cleaned e-commerce 
dataset of 1,200 orders.

---

## 🎯 Goal
Create visual representations of data to 
communicate insights clearly.

---

## ✅ Key Requirements Completed
- [x] Created charts (bar, line, pie/donut)
- [x] Chose appropriate visuals for each data type
- [x] Highlighted key insights
- [x] Built interactive dashboard with KPI cards
- [x] Added slicers for interactivity

---

## 🛠️ Tools Used
- Microsoft Power BI

---

## 📊 Visuals Created

| Visual | Chart Type | Insight |
|---|---|---|
| Monthly Revenue Trend | Line Chart | Revenue peaked Jun 2024 |
| Revenue by Product | Bar Chart | Chair & Printer lead |
| Order Status | Donut Chart | 41% cancelled/returned |
| Referral Source | Bar Chart | Instagram leads |
| Payment Method | Bar Chart | Online most popular |
| KPI Cards | Cards | Key metrics at a glance |

---

## 🎨 Dashboard Features
- **4 KPI Cards:** Total Orders, Total Revenue, 
  Avg Order Value, Total Outliers
- **2 Slicers:** Product filter, Order Status filter
- **5 Charts:** Line, Bar, Donut visualizations
- **Key Observations:** Written insights panel
- **Theme:** Professional olive green color scheme

---

## 💡 Key Business Insights Visualized
1. Total Revenue: **$1.26 million** across 1,200 orders
2. Chair & Printer top products at **~$195K each**
3. **41%** of orders cancelled or returned
4. **Instagram** drives most traffic (259 orders)
5. **8 outlier orders** detected above $3,330
6. **74%** of customers used coupon codes

---

## 💡 Key Learnings
- Choose chart types based on what story the data tells
- KPI cards give instant key metrics at a glance
- Consistent colors make dashboards look professional
- Slicers make dashboards interactive and dynamic
- Storytelling with data is as important as the data itself
